# 断言调试技能

## 使用场景
当断言失败但原因不明确时，帮助用户区分：
- **模型问题**：模型输出了错误的内容（需优化提示词）
- **断言问题**：断言配置过于严格或有误（需修改断言）
- **数据问题**：用例变量/期望值有误（需修改用例数据）

## 调试流程

1. **查看断言详情**：调用 `get_run_item_detail` 获取失败用例的 `assertion_details_json`
2. **对比分析**：
   - 模型实际输出 vs 断言期望值
   - 检查用例 variables_json 中的期望值是否正确
3. **分类判断**：
   - 模型输出合理但与期望不符 → 提示词问题
   - 模型输出格式正确但断言路径错误 → 断言配置问题
   - 期望值本身有误 → 数据问题

## 常见断言失败类型

| 类型 | 特征 | 解决方案 |
|------|------|----------|
| jsonPath 取值失败 | 路径不存在于输出中 | 检查输出结构，修正路径 |
| equals 不匹配 | 值不同但语义可能等价 | 考虑用 regex 或 inList |
| 输出非 JSON | contains/regex 失败 | 优化提示词强制 JSON 输出 |
| 变量引用错误 | equalsCaseVar/equalsSuiteVar | 检查变量名拼写 |

## 输出格式

```
**断言调试结果**：
- 失败类型：模型问题 / 断言配置问题 / 数据问题
- 原因分析：...
- 建议操作：修改提示词 / 修改断言 / 修改用例变量
```
