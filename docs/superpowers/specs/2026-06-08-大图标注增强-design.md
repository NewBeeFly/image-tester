# 大图标注浮窗增强：图片缩放、翻页、滚轮语义

日期：2026-06-08
状态：已批准
范围：仅前端（`web/src/components/CaseAnnotator.tsx`、`web/src/components/ImageViewer.tsx` 新增、`web/src/App.css`、`web/src/App.tsx`）

## 1. 背景

`CaseAnnotator`（提交 `f3c2a85` 引入）是数据页"大图标注"入口对应的浮窗，左大图 + 右可视化变量/断言。当前能力是按 `object-fit: contain` 居中显示，**不可缩放/不可平移/不可翻页/滚轮冲突**。

新需求：

1. 大图支持**缩放、平移、重置**。
2. 大图浮窗内**支持上一张 / 下一张**翻页。
3. 滚轮在图片区**用于缩放**，**不**穿透到外层列表。

不动后端 API、SchemaBuilder、VariableBuilder、AssertionBuilder。

## 2. 行为

### 2.1 缩放与平移

- 缩放范围 `0.25× ~ 8×`，按钮步进 `×1.25 / ÷1.25`。
- 右下角浮动工具条（4 项）：
  - `−`
  - 倍率显示（点击在 `100%` 与 `适应` 之间切换）
  - `+`
  - `适应`（重置）
- 在图片 stage 上滚轮 → 缩放，以光标点为锚点（光标下像素不动）。
- 拖动 → 平移；越界允许一段后回弹（不卡死）。
- 翻页/打开/关闭 → 重置为「适应」+ 居中。

### 2.2 翻页

- 范围：当前**数据页列表的可见顺序**（当前 `cases.map` 顺序，**不**重新排序、不依赖未来筛选/标签）。
- UI：图片 stage 左右两侧中点浮动 `‹` / `›` 半透明按钮：
  - `‹` 在 `currentIndex === 0` 时隐藏
  - `›` 在 `currentIndex === caseList.length - 1` 时隐藏
- **不**绑定键盘快捷键（用户确认不要键盘）。
- 翻页前**自动保存**当前用例的草稿（`onSave` 走现有 patch API `PATCH /api/test-cases/:id`）：
  - 保存失败 → 不切换，footer 上方红字提示。
  - 保存成功 → `await refreshCases(suiteId)` 拉最新列表 → 切换到 `cases[newIndex].id`（用 id 而非下标定位，避免数据刷新后下标漂移）。

### 2.3 滚轮语义

- **图片 stage 上**：`onWheel` 拦截 `preventDefault` → 缩放。
- **图片 stage 外**（即右侧 `.annotatorForm` 卡片内）：不拦截，沿用默认滚动。
- 这条边界解决了"滚轮滑动的是外层列表"的冲突。

### 2.4 加载与失败

- 每次 `src` 变 → `status = 'loading'`，stage 中央显示半透明 spinner。
- `img.onLoad` → `status = 'idle'`，触发内部 `resetView()`（用自然尺寸算 `fitScale`）。
- `img.onError` → `status = 'error'`，显示「加载失败 · 重试」按钮。点击 = 在 `src` query 追加 `?retry=${Date.now()}` 重新请求。

### 2.5 草稿 dirty 跟踪

`CaseAnnotator` 内部 `useRef<boolean>` 跟踪"是否有未保存修改"，对比 `relativePath / variablesJson / assertionsJson` 与 `testCase` 原始值。

## 3. 组件拆分

### 3.1 新增 `web/src/components/ImageViewer.tsx`

入参：

- `src: string`
- `alt: string`
- `onLoad?: () => void`
- `onError?: () => void`

内部状态：

- `scale: number`（> 0）
- `tx, ty: number`（CSS px）
- `status: 'idle' | 'loading' | 'error'`
- `naturalW, naturalH: number | null`

通过 `forwardRef` 暴露 `resetView(): void`，父级在翻页/打开/关闭时调用。

渲染结构：

```
<div class="ivStage" onWheel onPointerDown/Move/Up>
  <div class="ivImgWrap" style="transform: translate(tx,ty) scale(scale)">
    <img class="ivImg" src onLoad onError />
  </div>
  <button class="ivNav ivNavPrev" onClick={prev}>‹</button>
  <button class="ivNav ivNavNext" onClick={next}>›</button>
  <div class="ivToolbar">
    <button onClick={zoomOut}>−</button>
    <button onClick={toggleFit100}>{Math.round(scale*100)}%</button>
    <button onClick={zoomIn}>+</button>
    <button onClick={fit}>适应</button>
  </div>
  {status==='loading' && <div class="ivSpinner" />}
  {status==='error' && (
    <div class="ivError">
      <span>加载失败</span>
      <button onClick={retry}>重试</button>
    </div>
  )}
</div>
```

父级通过 `ref` 调用 `resetView()`；父级传入 `onPrev/onNext` 回调触发翻页（`ImageViewer` 不感知列表）。

### 3.2 改造 `web/src/components/CaseAnnotator.tsx`

新增 props：

- `caseList: TestCase[]`（已排序的可见列表）
- `currentIndex: number`
- `onRequestNavigate: (direction: -1 | 1) => Promise<void> | void`

行为：

- `<div class="annotatorImage">` 替换为 `<ImageViewer ref={ivRef} src={imageUrl(...)} alt={...} onPrev={() => void onRequestNavigate(-1)} onNext={() => void onRequestNavigate(1)} />`
- 在 `useEffect([open, testCase.id])` 中，开弹窗时 `ivRef.current?.resetView()`，并把 `dirty=false`
- 渲染时把 `‹` / `›` 禁用条件交给父级，父级在 `navigateCase` 内只切 id

### 3.3 改造 `web/src/App.tsx`

- 把 `annotatorCaseId` 切换包成 `navigateCase(direction)`：
  1. 取得 `currentIndex`（若 `annotatorCaseId` 不在 `cases` 中则直接 return，不翻）
  2. 计算 `nextIndex = clamp(currentIndex + direction, 0, cases.length - 1)`
  3. 如果 `currentIndex === nextIndex`（即无上/下一张）return
  4. **如果草稿 dirty** → `await saveCurrentAnnotator()`（包装现有 `PATCH` 逻辑）
  5. `await refreshCases(dataSuiteId)`
  6. 用 `cases.find(c => c.id === cases[nextIndex].id)?.id` 取新 id
  7. `setAnnotatorCaseId(newId)`
- 把 `caseList={cases} currentIndex={currentIndex} onRequestNavigate={(d) => void navigateCase(d)}` 透传给 `CaseAnnotator`
- 现有 Esc / 点遮罩 / 取消按钮 → 走 `onClose`，不触发自动保存（保留用户点"保存"显式行为）

## 4. 样式（`web/src/App.css`）

新增类（不重写现有 `.annotatorImage` 的尺寸约束，只在其内部嵌套一层）：

- `.ivStage`：`position: relative; width: 100%; height: 100%; overflow: hidden; cursor: grab;`，`touch-action: none`（阻止移动端默认手势）
- `.ivStage:active`：`cursor: grabbing`
- `.ivImgWrap`：`position: absolute; left: 0; top: 0; transform-origin: 0 0; will-change: transform; pointer-events: none; image-rendering: auto`
- `.ivImg`：`display: block; pointer-events: none; user-select: none; -webkit-user-drag: none`
- `.ivToolbar`：`position: absolute; right: 12px; bottom: 12px; display: flex; gap: 6px; background: rgba(15,23,42,0.78); border-radius: 8px; padding: 4px; color: #fff`
- `.ivToolbar button`：`background: transparent; color: inherit; border: 0; padding: 4px 8px; border-radius: 4px; font-size: 12px; cursor: pointer`
- `.ivToolbar button:hover`：`background: rgba(255,255,255,0.12)`
- `.ivNav`：`position: absolute; top: 50%; transform: translateY(-50%); width: 36px; height: 36px; border-radius: 50%; background: rgba(15,23,42,0.7); color: #fff; border: 0; font-size: 22px; line-height: 1; cursor: pointer; opacity: 0; transition: opacity 0.15s`
- `.ivStage:hover .ivNav`：`opacity: 0.9`
- `.ivNavPrev`：`left: 12px`
- `.ivNavNext`：`right: 12px`
- `.ivSpinner`：`position: absolute; inset: 0; display: flex; align-items: center; justify-content: center; background: rgba(15,23,42,0.4); color: #fff; font-size: 12px`
- `.ivError`：同 spinner 区域布局，按钮 `margin-left: 8px`

`fit` 时 `ivImg` 用 `width: 100%; height: 100%; object-fit: contain`（`<img>` 自然行为），不再依赖 `transform: scale`；具体实现：`fitScale === 1 && tx === 0 && ty === 0` 时切到一个 CSS class `.ivImgFit`，其他时候用 transform 路径。`scale=1` 与"原图 100% 大小"可能不同，工具条点击倍率在 `100%` 与 `fit` 之间切换 → 内部判断走两条路径。

## 5. 边界场景

| 场景 | 行为 |
| --- | --- |
| 网络断开切下一张 | 错误占位 + 重试按钮 |
| 保存失败翻页 | 不切换，红字提示，停留在当前用例 |
| 关闭弹窗 | Esc / 遮罩 / 取消按钮均**不**触发自动保存 |
| 滚轮在右侧表单 | 不拦截，正常滚动 |
| 同一 id 重复打开 | `useEffect([open, testCase.id])` 重新初始化 |
| 保存后下标漂移 | 切下一张时用 `cases.find(id)` 取新 id 而非按下标 |
| 缩放超过 8× | clamp 到 8 |
| 缩放小于 0.25× | clamp 到 0.25 |
| 移动端 <960px | 沿用现有 media query（图片区 40vh），stage 自适应 |

## 6. 测试

项目无现成单测基建，本组件与现有 `CaseAnnotator` 一致不做单测；用浏览器手工验证：

1. 打开大图标注 → 图片「适应」状态
2. 工具条 `+`/`−` → 倍率正确显示，最大 800%，最小 25%
3. 倍率按钮 → 在 100% / 适应 切换
4. 滚轮（图片上）→ 缩放，光标下像素不动；表单上滚轮 → 滚动表单
5. 放大后拖动 → 平移；越界回弹
6. 翻页 → 缩放重置为适应
7. 翻页前 dirty → 自动保存后切换；保存失败 → 不切换
8. 网络断开翻页 → 错误占位 + 重试
9. Esc / 遮罩 / 取消 → 关闭弹窗不自动保存
10. 回归：变量/断言可视化编辑器仍可用

## 7. 风险

- 滚轮在 stage 上的 `preventDefault` 在某些极旧浏览器（已不维护）可能不生效；项目目标是现代浏览器，不处理。
- `transform: scale` 8× 时浏览器插值可能糊；不引入 `image-rendering: pixelated`，避免过锐失真。如后续需做"放大到看清微小文字"，再迭代。
- 自动保存依赖网络；网络抖动时翻页慢。已在保存失败时给出反馈，UX 可接受。
